# RM-012G MCP Server

An advanced Model Context Protocol (MCP) server for controlling and monitoring **RM-012G** devices via their REST API. This server enables LLMs (like Gemini or Claude) to interact directly with industrial control systems, perform network scans, manage VPNs, and more.

## ✨ Key Features

- **Industrial Control**: Real-time Modbus tag reading/writing.
- **System Management**: Monitor system health, network interfaces, and processes.
- **Service Control**: Manage Node-RED, MQTT Broker, and VPN services.
- **Data Access**: List and read CSV data files generated by the device.
- **Network Tools**: Perform network scans to discover Modbus devices.

## 🚀 Setup

### 1. Prerequisites
- [Node.js](https://nodejs.org/) (v16.x or later)
- Access to an RM-012G device API.

### 2. Configuration
Create a `.env` file in this directory:

```env
RM012G_API_URL=http://your-device-ip:8080
RM012G_API_TOKEN=your_jwt_token_here (optional)
```

### 3. Integration
Add the server to your `mcp-config.json` (e.g., in VS Code or Claude Desktop):

```json
{
  "mcpServers": {
    "rm012g": {
      "command": "node",
      "args": ["D:/antigravity/MCP-Server/rm-012g/index.js"],
      "env": {
        "RM012G_API_URL": "http://your-device-ip:8080"
      }
    }
  }
}
```

## 🛠 Available Tools

| Category | Tools |
| :--- | :--- |
| **Authentication** | `login` |
| **System** | `get_system_status`, `get_network_status`, `scan_network` |
| **Modbus** | `get_devices`, `get_tags`, `write_tag` |
| **Services** | `get_mqtt_status`, `control_mqtt_service`, `get_node_red_status`, `control_node_red` |
| **Files** | `list_csv_files`, `read_csv_file` |
| **Networking** | `get_vpn_status`, `control_vpn` |
